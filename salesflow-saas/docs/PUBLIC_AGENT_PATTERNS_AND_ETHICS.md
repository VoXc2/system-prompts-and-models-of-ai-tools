# أنماط الوكلاء العامة والالتزام الأخلاقي (2026)

## 1) لماذا لا نعتمد «التسريبات»

- محتوى **مسروق أو داخلي** (تسريبات شركات، بوتات سحابية، حزم داخلية) قد يكون **سرّياً** أو **غير دقيق** أو **غير قانوني** الاستخدام.
- **لا** نُدخل في المنتج سلوكاً مبنياً على وثائق مسربة؛ نلتزم **المصادر العامة**: مدونات رسمية، وثائق منتج، أبحاث وبيانات صناعية.

## 2) ما هو معروف علنا (مرجع عام — ليس سراً)

| المصدر (عام) | الفكرة | ارتباط بـ Dealix |
|--------------|--------|------------------|
| Cursor — Cloud Agents | وكلاء يعملون في السحابة، مهام طويلة، تكامل Slack/GitHub | نمط: **مهام خلفية + مراقبة** → Celery، `agent_tasks` |
| Cursor — Agent mode | تنفيذ متعدد الملفات مع أدوات | نمط: **AgentExecutor + أدوات** |
| ممارسات Google Cloud / OpenAI Codex (عام) | تخطيط، قيود، معايير إنجاز، اختبارات | نمط: **حلقات تحسين + بوابات إطلاق** |
| Spring AI / أنماط «recursive advisors» (عام) | حلقات إعادة محاولة وتحقق | نمط: **`self_improvement_flow` + checkpoints** |

روابط رسمية للاطلاع (للمطورين): [Cursor Cloud Agents](https://cursor.com/docs), [Cursor Features](https://cursor.com/features).

## 3) ما طبّقناه في المستودع (بدون تسريبات)

- **إشارات تطور حقيقية** تُحقَن دورياً في حلقة التحسين: لقطة من `agent_framework_report` (إصدارات LangGraph، AutoGen، Celery، …).
- **نقطة مراقبة**: `GET /api/v1/agent-frameworks`.
- **سجل أتمتة**: `app/ai/automation/registry.py`.
- **مهمة Celery**: `snapshot_agent_framework_stack`.

## 4) الخطوة التالية المشروعة

- RAG داخلي على **وثائقكم** فقط (ليس تسريبات خارجية).
- Langfuse / OpenTelemetry لمراقبة **جودة وتكلفة** الاستدعاءات.
- اختبارات عقد ذهبية لكل وكيل حرج.
