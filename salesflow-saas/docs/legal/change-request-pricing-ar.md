## 1. الغرض

تحدد هذه السياسة تعريف طلبات التعديل، أنواعها، تسعيرها، وإجراءات تقديمها والموافقة عليها وتنفيذها. تُطبق هذه السياسة على جميع العملاء الحاليين لمنصة Dealix (ديل اي اكس).

## 2. تعريف طلب التعديل

### 2.1 ما هو طلب التعديل؟

طلب التعديل (Change Request) هو أي طلب من العميل لإجراء تغيير على المنصة أو الخدمات المُقدمة يقع خارج نطاق العمل المتفق عليه في العقد الأصلي أو أمر العمل (SOW) أو يتجاوز ساعات التعديل المجانية المشمولة في خطة الصيانة.

### 2.2 ما يُعتبر طلب تعديل

- إضافة ميزات أو وظائف جديدة لم تكن ضمن النطاق الأصلي
- تعديل سير العمل أو العمليات المؤتمتة بعد الموافقة النهائية
- تغيير التصميم أو واجهة المستخدم بعد اعتماد التصميم
- إضافة تكاملات جديدة مع أنظمة خارجية
- تعديل التقارير المخصصة أو إضافة تقارير جديدة
- تغيير هيكل البيانات أو إضافة حقول مخصصة (بعد الاعتماد)
- طلب تدريب إضافي يتجاوز ما هو مشمول في الخطة
- أي عمل يتجاوز الساعات المجانية في خطة الصيانة

### 2.3 ما لا يُعتبر طلب تعديل

- إصلاح الأعطال والأخطاء في النظام (bugs)
- تحديثات الأمان والنظام الدورية
- التعديلات المشمولة في ساعات الصيانة المجانية
- استفسارات الدعم الفني العادية
- إعادة تعيين كلمات المرور وإدارة المستخدمين الأساسية

## 3. أنواع التعديلات والتسعير

### 3.1 السعر القياسي

**السعر الأساسي: 150 ريال سعودي / ساعة**

### 3.2 تصنيف التعديلات

#### النوع الأول: تعديل بسيط

| البند | التفاصيل |
|-------|----------|
| **الوصف** | تعديلات طفيفة لا تتطلب تطويراً برمجياً معقداً |
| **الوقت المقدر** | 1 - 3 ساعات |
| **نطاق التكلفة** | 150 - 450 ر.س |
| **وقت التنفيذ** | 1 - 3 أيام عمل |

**أمثلة:**
- تعديل النصوص والعناوين في الواجهات
- تغيير الألوان أو الخطوط
- تعديل إعدادات الإشعارات
- تحديث قوالب الرسائل
- إضافة أو تعديل حقول بسيطة
- تعديل صلاحيات المستخدمين
- تعديل جداول المواعيد

#### النوع الثاني: تعديل متوسط

| البند | التفاصيل |
|-------|----------|
| **الوصف** | تعديلات تتطلب تطويراً برمجياً متوسطاً أو تغييرات في سير العمل |
| **الوقت المقدر** | 4 - 12 ساعة |
| **نطاق التكلفة** | 600 - 1,800 ر.س |
| **وقت التنفيذ** | 3 - 7 أيام عمل |

**أمثلة:**
- إضافة حقول مخصصة متعددة مع منطق مشروط
- تعديل سير العمل الآلي (إضافة خطوات، تغيير شروط)
- إنشاء تقارير مخصصة جديدة
- تعديلات في تصميم صفحات محددة
- إضافة قوالب بريد إلكتروني أو رسائل مخصصة
- تهيئة أتمتة جديدة بسيطة
- تعديل نماذج الإدخال

#### النوع الثالث: تعديل معقد

| البند | التفاصيل |
|-------|----------|
| **الوصف** | تعديلات جوهرية تتطلب تطويراً برمجياً مكثفاً أو تكامل مع أنظمة خارجية |
| **الوقت المقدر** | 13+ ساعة |
| **نطاق التكلفة** | 1,950+ ر.س |
| **وقت التنفيذ** | 1 - 4 أسابيع |

**أمثلة:**
- تكامل مع نظام خارجي (ERP, CRM آخر, بوابة دفع)
- تطوير ميزة جديدة بالكامل
- إعادة تصميم قسم كامل من الواجهة
- تطوير لوحة معلومات مخصصة
- إنشاء سير عمل آلي معقد متعدد المراحل
- تطوير تكامل API مخصص
- ترحيل بيانات من نظام آخر

### 3.3 جدول الأسعار حسب نوع العمل

| نوع العمل | السعر (ر.س/ساعة) | ملاحظات |
|----------|-------------------|---------|
| تعديلات وتهيئة عامة | 150 | السعر القياسي |
| تصميم واجهات | 175 | يشمل تصميم UI/UX |
| تطوير برمجي | 200 | تطوير ميزات جديدة |
| تكامل مع أنظمة خارجية | 225 | يشمل API والاختبار |
| استشارات تقنية | 250 | تحليل وتوصيات |
| عمل مستعجل (خارج ساعات العمل) | السعر القياسي × 1.5 | عند طلب التنفيذ المستعجل |
| عمل في العطل الرسمية | السعر القياسي × 2 | عند طلب التنفيذ في العطل |

### 3.4 الحد الأدنى للفوترة

- الحد الأدنى لأي طلب تعديل: **30 دقيقة** (75 ر.س).
- تُحتسب الأجزاء بعد أول 30 دقيقة بوحدات **15 دقيقة**.

## 4. عملية طلب التعديل

### 4.1 الخطوة 1: تقديم الطلب

يقدم العميل طلب التعديل عبر إحدى القنوات التالية:
- **نموذج طلب التعديل الإلكتروني** (الطريقة المفضلة)
- **البريد الإلكتروني**: changes@dealix.sa
- **مدير الحساب المخصص** (للخطة المميزة)

يجب أن يتضمن الطلب:

| البند | مطلوب/اختياري |
|-------|---------------|
| وصف التعديل المطلوب | مطلوب |
| السبب / الحاجة التجارية | مطلوب |
| الأولوية (عاجل / عادي / منخفض) | مطلوب |
| الموعد المطلوب للتنفيذ | اختياري |
| لقطات شاشة أو رسوم توضيحية | اختياري (موصى به) |
| مرجع أمر العمل الأصلي | اختياري |

### 4.2 الخطوة 2: التقييم والتسعير

1. يراجع الفريق التقني الطلب خلال **(2) أيام عمل**.
2. يتم تصنيف التعديل (بسيط / متوسط / معقد).
3. يُعد تقدير تفصيلي يشمل:
   - نطاق العمل المطلوب
   - عدد الساعات المقدرة
   - التكلفة الإجمالية
   - الجدول الزمني المتوقع
   - أي تأثير على الخدمات الحالية
4. يُرسل التقدير للعميل عبر البريد الإلكتروني.

### 4.3 الخطوة 3: الموافقة

| مبلغ التعديل | إجراء الموافقة |
|-------------|----------------|
| أقل من 500 ر.س | موافقة عبر البريد الإلكتروني |
| 500 - 2,000 ر.س | موافقة كتابية عبر نموذج الموافقة |
| أكثر من 2,000 ر.س | أمر عمل (SOW) منفصل موقع من الطرفين |

**مدة صلاحية التقدير**: (15) يوماً من تاريخ الإصدار.

**شروط الموافقة**:
- يجب أن تكون الموافقة من شخص مخول (منسق المشروع أو الإدارة).
- الموافقة تعني الالتزام بالتكلفة المحددة في التقدير.
- لا يبدأ العمل إلا بعد الحصول على الموافقة الرسمية.

### 4.4 الخطوة 4: التنفيذ

1. يتم جدولة العمل وفقاً للأولوية والموارد المتاحة.
2. يُعيّن مسؤول تنفيذ للطلب.
3. يتم إشعار العميل ببدء العمل.
4. يتم تقديم تحديثات دورية حسب حجم التعديل:
   - بسيط: إشعار عند الاكتمال
   - متوسط: تحديث واحد خلال التنفيذ + إشعار الاكتمال
   - معقد: تحديثات أسبوعية + نقاط مراجعة

### 4.5 الخطوة 5: الاختبار والتسليم

1. يتم اختبار التعديل داخلياً.
2. يُنشر التعديل في بيئة الاختبار (للتعديلات المتوسطة والمعقدة).
3. يراجع العميل التعديل ويقدم ملاحظاته خلال **(3) أيام عمل**.
4. يتم معالجة الملاحظات (إن وجدت) ضمن جولة مراجعة واحدة مجانية.
5. يُنشر التعديل في بيئة الإنتاج بعد الموافقة.
6. يتم إرسال تقرير التسليم والفاتورة.

### 4.6 الخطوة 6: الفوترة

- تُصدر الفاتورة بعد التسليم والقبول.
- مدة السداد: (15) يوماً من تاريخ الفاتورة.
- يتم تضمين ضريبة القيمة المضافة (15%).

## 5. ما يُعتبر خارج النطاق

البنود التالية لا تُعالج كطلبات تعديل عادية وتتطلب مشروعاً منفصلاً:

### 5.1 مشاريع التطوير الكبرى

- بناء وحدات أو أنظمة جديدة بالكامل
- إعادة تصميم شامل للمنصة
- تطوير تطبيق جوال مستقل
- ترحيل كامل من نظام إلى آخر

### 5.2 متطلبات خاصة

- تطوير تكاملات معقدة تتطلب أكثر من 40 ساعة
- تعديلات تتطلب تغيير البنية التحتية
- متطلبات أمنية أو تنظيمية تتطلب شهادات خاصة
- تعديلات تؤثر على جميع عملاء المنصة

### 5.3 الإجراء للمشاريع الكبرى

1. يتم تقييم الطلب كمشروع مستقل.
2. يُعد اقتراح مشروع شامل يتضمن النطاق والجدول والتكلفة.
3. يُوقع أمر عمل (SOW) منفصل.
4. يُطبق جدول دفعات المشاريع (مقدم + مراحل + نهائي).

## 6. سياسة الإلغاء والتعديل

### 6.1 إلغاء طلب التعديل

| مرحلة الإلغاء | الرسوم |
|-------------|--------|
| قبل بدء العمل | بدون رسوم |
| بعد بدء العمل وقبل اكتماله | تُحتسب الساعات الفعلية المنفذة |
| بعد الاكتمال وقبل النشر | تُحتسب كامل التكلفة |

### 6.2 تعديل طلب التعديل

- يمكن تعديل نطاق الطلب قبل بدء التنفيذ دون رسوم إضافية.
- التعديلات بعد بدء التنفيذ قد تؤثر على التكلفة والجدول الزمني.
- يتم إصدار تقدير معدل للموافقة عليه.
