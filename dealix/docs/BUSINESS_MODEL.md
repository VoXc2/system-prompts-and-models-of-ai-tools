# نموذج الأعمال — Dealix (مستند استراتيجي شامل)

## ما هو Dealix؟

Dealix **نظام إيرادات B2B سعودي (Saudi Revenue OS)** يجمع: **إشارات سوق (لماذا الآن)**، **ذاكرة إيرادات وذاكرة مشروع**، **مشغّل استراتيجي عربي** يومي، **وكلاء تنفيذ بموافقة**، **امتثال واتساب/اتصال**، و**حزمة إثبات عائد (Proof Pack)**. الهدف ليس «تخزين جهات اتصال» بل **تحويل السياق إلى قرارات ومسودات ومواعيد وإيرادات — بأمان**.

## لماذا ليس CRM؟

الـ CRM يركز على السجلات والأنابيب. Dealix يركز على **السياق (سوق + علاقة + مشروع)** و**القرار اليومي** و**المسار الآمن للتنفيذ**. يمكن أن يعيش **فوق** CRM أو بجانبه؛ لا يُباع كبديل عميق لـ Salesforce/HubSpot في كل وظائفهما.

## لماذا ليس «مرسل واتساب»؟

لأن الإرسال الجماعي البارد يعرّض العلامة والامتثال للخطر. Dealix يصمم **مسودات + موافقة + سجل + سياسات**؛ الواتساب **قناة أولى في GCC** لكن **ليست قناة إزعاج**.

## 1. الملخص التنفيذي

فئة جديدة: Revenue OS سعودي سياقي + مشغّل عربي + حوكمة تنفيذ.

## 2. فئة المنتج

Saudi B2B Revenue OS + Arabic Personal Strategic Operator + Market Radar + Compliance OS + Revenue Memory + Project Intelligence.

## 3. ICP (العميل المثالي)

- مؤسسو B2B في السعودية/الخليج يعتمدون واتساب في المتابعة.
- شركات صغيرة/متوسطة بفريق مبيعات 1–15 يحتاجون إثباتاً للإدارة.
- وكالات نمو تبحث عن تموضع تقني/عربي متميز.

## 4. شخصيات المشتري (Buyer personas)

| الشخصية | الدافع | مخاوف |
|---------|--------|--------|
| المؤسس | سرعة، تركيز، علاقات | التشتت، سمعة العلامة |
| مدير المبيعات | الأنابيب والاجتماعات | جودة الرسائل، الامتثال |
| العمليات/الامتثال | السجل والسياسات | واتساب بارد، PDPL |
| شريك وكالة | هامش وهوية | تعقيد التنفيذ |

## 5. المشكلة (Pain)

- أدوات كثيرة بلا سياق موحّد للسوق السعودي.
- صعوبة إثبات ROI للإدارة.
- مخاطر تواصل (واتساب/بريد) بدون موافقة وتدقيق.
- ضعف الصياغة العربية الاستراتيجية «اللي تبيع» بدون مبالغة.

## 6. الحل (Solution)

طبقة Dealix: رادار + فرص + مسودات عربية + قبول/تخطي + مسودات اجتماع/بريد + ذاكرة مشروع/إيرادات + تقارير جاهزية.

## 7. القيمة (Value)

«يقلل وقت القرار، يرفع جودة التواصل، يمنع المخاطر، ويربط النشاط بالإيرادات عبر Proof Pack.»

## 8. لماذا الآن؟

سوق 2026: انتقال من copilots عامة إلى **وكلاء متخصصين بسياق**؛ توسّع أنظمة «Revenue» خارج تسجيل المكالمات فقط؛ طلب واضح على **عربي + امتثال GCC + إثبات**.

## 9. الفرق عن المنافسين (ملخص)

| فئة | أمثلة | ملخص موقف Dealix |
|-----|--------|-------------------|
| منصات واسعة | HubSpot, Salesforce | سياق سعودي + مشغّل عربي + موافقات واتساب أصلية في المنتج |
| ذكاء إيرادات | Gong | واتساب + سوق محلي + مسار علاقة→إيراد |
| بيانات/توسع | Apollo, ZoomInfo | لا نسخ بارد؛ تأهيل + قمع + موافقة |
| suites | Zoho, Odoo | لسنا ERP؛ Revenue OS فوق الأدوات |
| أدوات واتساب | أتمتة عامة | لا إرسال بارد؛ opt-in + أزرار + تدقيق |
| وكالات | — | الشراكة: تنفيذ + اشتراك منصة |
| AI عام | وكلاء عامون | الحوكمة والذاكرة جزء من المنتج |
| Boardy-like | intros | علاقة + إيراد + ذاكرة مشروع + امتثال |
| SocraticCode-like | فهرسة كود | + استراتيجية وGTM وليس كود فقط |

التفاصيل: `docs/COMPETITIVE_POSITIONING.md` و`GET /api/v1/business/competitors`.

## 10. الـ Moat (الخندق)

1. سياق سعودي/B2B.  
2. عربي أولاً في المشغّل والمسودات.  
3. واتساب أولاً مع تصميم امتثال.  
4. إشارات «لماذا الآن».  
5. Revenue Memory + Project Intelligence.  
6. بوابات موافقة وتتبع.  
7. Proof Pack شهري.  
8. Playbooks عمودية.  
9. شراكات وكالة منظمة.  
10. بيانات وتعلم من Accept/Skip (مع حدود خصوصية).

## 11–12. مصادر الإيراد وهيكل التكلفة

**إيراد:** اشتراكات (Founder / Growth / Scale)، إعداد وكالة، أداء (عقدي)، مؤسسات مخصصة.  
**تكلفة:** LLM، بنية (Supabase/Compute)، دعم ونجاح عملاء، تسويق، شراكات، امتثال قانوني.

## 21. المستشارون والمستثمرون

لا مقدمات مدفوعة قبل إطار قانوني وتتبع قيمة واضح.

## 22. Marketplace (مستقبل)

## 23–25. خنادق البيانات والAI والسوق
