# سياسة احتفاظ بالبيانات — Dealix (مسودة تشغيلية)

| نوع البيانات | فترة احتفاظ افتراضية | ملاحظات |
|--------------|----------------------|---------|
| بيانات حساب pilot نشط | مدة العقد + 30 يوماً للتصدير | يمكن التمديد للنزاعات بقرار قانوني |
| Project memory / embeddings | مدة العقد أو حتى طلب الحذف | بعد الإنهاء: حذف صفوف المشروع في Supabase وفق [`SUPABASE_STAGING_RUNBOOK.md`](SUPABASE_STAGING_RUNBOOK.md) |
| سجلات تطبيق (structured) | 30–90 يوماً على staging؛ الإنتاج حسب التكلفة | قلل الحقول الشخصية؛ استخدم معرفات داخلية |
| Sentry events | حسب خطة Sentry | تجنب إرفاق bodies كاملة |
| Langfuse traces | حسب خطة Langfuse | لا تضع مفاتيح API أو محتوى عميل كامل |
| Webhooks DLQ | 7–30 يوماً | إعادة معالجة ثم مسح |
| فواتير / مدفوعات | حسب المحاسب والزكاة | لا تحذف سجلات ضريبية دون توجيه محاسبي |

## الإجراء عند طلب الحذف

1. تحقق هوية الطالب والصلاحية.
2. حدد النطاق (شركة، مستخدم، مشروع).
3. نفّذ الحذف في DB + Supabase + مسح فهارس embeddings ذات الصلة.
4. أكّد كتابياً مع SLA داخلي.

## المراجعة الدورية

مراجعة كل 6 أشهر: ما ما زال ضرورياً، وتقليل الحقول المخزنة.
